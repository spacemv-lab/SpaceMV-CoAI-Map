# 架构设计（map-ai）

## 目标与风格

- 面向“大屏科技风”，默认深色主题，强调信息密度与可读性
- 布局参考“若依”后台的导航范式：顶部横向导航
- 顶部结构：左侧 Logo 与标题；中间主菜单；右侧账号区域（暂留占位）
- 地图与可视化组件占据主画布，全屏承载交互，顶部导航悬浮/透明玻璃风格

## 技术栈选型

- 前端：React 18 + Vite 5
- 路由：React Router v6（嵌套路由 + 懒加载）
- 样式：Tailwind CSS + CSS 变量（配合现有 shadcn/ui 组件）
- UI 组件：shadcn/ui（Radix UI 基座）+ lucide-react 图标
- 状态管理：轻量优先，推荐 Zustand（或直接以组件本地状态为主）
- 图形/地图：Cesium（已集成），保留 MapViewer 作为主画布
- 构建：Nx + Vite（已配置），单体 app 采用按需/动态导入优化

## 路由与布局

- 路由结构
  - 根路由：/（挂载 RootLayout）
  - 子路由：/home、/analysis、/tasks、/settings、/about（示例，后续可裁剪）
  - 懒加载：页面以动态 import 方式加载，hover 预取
- 布局结构（顶部栏）
  - Left：Logo + Title（可点击返回首页）
  - Center：一级菜单（横向，居中，支持高亮与下划线指示）
  - Right：账号区（占位，后续接入登录/头像/下拉）
- 主内容区：地图画布及业务组件，位于导航之下；顶部栏采用半透明玻璃化+模糊效果，避免遮挡信息
- Z 轴分层：MapViewer 为底层（z-0），聊天/面板悬浮（z-10），导航（z-20）

## 菜单与导航策略

- 菜单来源：静态配置（后续可改为服务端下发）
- 高亮规则：基于 pathname 精确匹配，支持别名/重定向
- 懒加载优化：使用 React.lazy + suspense；对主路径在首屏预取
- 动态导入命名块：确保 chunk 命名可读，利于调试与分析

## 状态管理

- 轻量原则：不引入全局复杂状态库，优先组件本地状态 + 简单 store
- 建议引入 Zustand 管理跨页面 UI 状态（如主题、菜单活跃项、面板开关）
- 地图相关状态由 MapViewer/特性模块自主管理，避免全局耦合

## 主题与样式

- 基础：Tailwind + CSS 变量（--color-primary、--glass-bg 等）
- 深色默认：配合“科技风”霓虹强调色（如蓝青紫）
- 玻璃效果：backdrop-filter + 半透明边框；注意性能与清晰度
- 响应式：顶部栏在窄屏自动折叠为汉堡菜单（后续需求驱动）

## 性能与工程实践

- 动态导入页面与重型组件，降低首屏体积
- 预加载：菜单 hover/focus 时预取对应路由 chunk
- 避免渲染瀑布：对独立数据请求使用 Promise.all
- 组件拆分：将昂贵组件独立并 memo；减少不必要 re-render
- 资源：Cesium 静态资源已通过 viteStaticCopy 配置
