## 1. 架构概述

### 1.1 设计目标

| 目标 | 描述 | 状态 |
|------|------|------|
| 二三维切换 | 支持 2D 点（PointPrimitive）和 3D 广告牌（Billboard）动态切换 | ✅ |
| 自定义符号 | 支持圆形、方形、三角形、五角星等 Canvas 生成符号 | ✅ |
| 图片上传 | 支持用户上传自定义图标（PNG/JPG/SVG） | ✅ |
| 文字标注 | 支持属性字段绑定、动态字体大小、距离调节 | ✅ |
| 高度偏移 | 支持点的高度调节，解决贴地/悬空需求 | ✅ |
| 数据回写 | GeoJSON 数据加载后回写 store，供渲染器使用 | ✅ |

### 1.2 架构分层

## 4. 核心组件实现

### 4.1 PointRendererManager

**位置**: `renderer/point-renderer.ts`

**职责**: 渲染器调度器，根据 `pointRenderMode` 动态选择渲染策略

```typescript
export class PointRendererManager implements IPointRenderer {
  private currentRenderer: IPointRenderer | null = null;
  private currentMode: PointRenderMode | null = null;
  private layerId: string | null = null;

  async update(layer: LayerState, viewer: Cesium.Viewer): Promise<void> {
    const mode = layer.style.pointRenderMode || 'billboard';

    // 检查是否需要切换渲染器
    if (this.currentMode !== mode || this.layerId !== layer.id) {
      // 销毁旧渲染器
      if (this.currentRenderer) {
        this.currentRenderer.destroy();
        this.currentRenderer = null;
      }

      // 创建新渲染器
      this.currentRenderer = createPointRenderer(mode);
      this.currentMode = mode;
      this.layerId = layer.id;
    }

    // 委托给当前渲染器
    if (this.currentRenderer instanceof BillboardRenderer) {
      await this.currentRenderer.update(layer, viewer);
    } else if (this.currentRenderer) {
      this.currentRenderer.update(layer, viewer);
    }
  }

  destroy(): void {
    if (this.currentRenderer) {
      this.currentRenderer.destroy();
      this.currentRenderer = null;
    }
    this.currentMode = null;
    this.layerId = null;
  }
}
```

### 4.2 BillboardRenderer

**位置**: `renderer/billboard-renderer.ts`

**特点**:
- 使用独立的 `BillboardCollection` 和 `LabelCollection`（不通过 Entity）
- 支持图片上传和 Canvas 符号生成
- Label 位置锁定在 Billboard 正上方，距离可调

**关键实现**:

```typescript
async update(layer: LayerState, viewer: Cesium.Viewer): Promise<void> {
  // 1. 检查是否需要重建 collection
  if (!this.billboardCollection || this.layerId !== layer.id) {
    this.destroy();
    this.layerId = layer.id;
    this.billboardCollection = viewer.scene.primitives.add(
      new Cesium.BillboardCollection()
    );
  }

  // 2. 处理标签集合
  const hasLabels = !!layer.style.label?.text;
  if (hasLabels && !this.labelCollection) {
    this.labelCollection = viewer.scene.primitives.add(
      new Cesium.LabelCollection()
    );
  } else if (!hasLabels && this.labelCollection) {
    viewer.scene.primitives.remove(this.labelCollection);
    this.labelCollection = null;
    this.labelIdMap.clear();
  }

  // 3. 清空现有（防止重叠）
  if (this.billboardCollection) {
    this.billboardCollection.removeAll();
  }
  if (this.labelCollection) {
    this.labelCollection.removeAll();  // ← 关键：必须清空
  }
  this.featureIdMap.clear();
  this.labelIdMap.clear();

  // 4-6. 添加 billboard 和 label
  // ...
}

private addLabel(
  feature: any,
  layer: LayerState,
  override?: Partial<LabelStyle>,
  finalHeight?: number,
): Cesium.Label {
  // 解析属性字段值：{fieldName} -> actualValue
  const rawText = override?.text || labelStyle.text || '';
  const text = rawText.replace(/\{(\w+)\}/g, (match, fieldName) => {
    return feature.properties?.[fieldName] || match;
  });

  // 使用用户配置的距离
  const labelOffset = layer.style.label?.labelOffset ?? 10;

  return this.labelCollection!.add({
    position: Cesium.Cartesian3.fromDegrees(lng, lat, finalHeight || height),
    text: text,
    font: override?.font || labelStyle.font || '14px sans-serif',
    horizontalOrigin: Cesium.HorizontalOrigin.CENTER,
    verticalOrigin: Cesium.VerticalOrigin.BOTTOM,
    pixelOffset: new Cesium.Cartesian2(0, -labelOffset),  // ← 负值向上
  });
}
```

## 6. 关键技术决策

### 6.1 为什么不使用 Entity.billboardCollection？

**问题**: Cesium 的 `Entity` 类没有 `billboardCollection` 属性，只有单个 `billboard` 属性（`BillboardGraphics` 类型）。

**解决方案**: 直接使用独立的 `BillboardCollection`，添加到 `scene.primitives`：

```typescript
// ❌ 错误
this.entity = new Cesium.Entity();
this.billboardCollection = this.entity.billboardCollection;  // undefined!

// ✅ 正确
this.billboardCollection = viewer.scene.primitives.add(
  new Cesium.BillboardCollection()
);
```

### 6.2 为什么移除 heightReference？

**问题**: 当 `BillboardCollection` / `PointPrimitiveCollection` 直接添加到 `scene.primitives` 时，不支持 `heightReference` 参数，会报错：
```
DeveloperError: Height reference is not supported without a scene.
```

**解决方案**: 移除 `heightReference`，使用 `pointHeightOffset` 手动控制高度：

```typescript
// ❌ 错误
const billboard = this.billboardCollection.add({
  position: Cesium.Cartesian3.fromDegrees(lng, lat, height),
  heightReference: Cesium.HeightReference.CLAMP_TO_GROUND,  // 报错!
});

// ✅ 正确
const heightOffset = layer.style.pointHeightOffset ?? 0;
const finalHeight = height + heightOffset;

const billboard = this.billboardCollection.add({
  position: Cesium.Cartesian3.fromDegrees(lng, lat, finalHeight),
  // 不使用 heightReference
});
```

### 6.3 为什么 LabelCollection 必须清空？

**问题**: 每次 `update` 调用时，如果只清空 `billboardCollection` 而不清空 `labelCollection`，会导致 Label 重复添加，产生重叠。

**解决方案**: 同时清空两个 collection：

```typescript
// ✅ 正确
if (this.billboardCollection) {
  this.billboardCollection.removeAll();
}
if (this.labelCollection) {
  this.labelCollection.removeAll();  // ← 必须！
}
this.featureIdMap.clear();
this.labelIdMap.clear();
```

### 6.4 数据为什么回写 store？

**问题**: 最初 GeoJSON 数据只用于创建 DataSource，没有保存，导致 renderer 无法从 `layer.data` 读取。

**解决方案**: 调用 `updateLayerData` 将数据回写 store：

```typescript
const loadGeoJsonData = async (layer: LayerState): Promise<void> => {
  const response = await fetch(`/api/datasets/${layer.sourceId}/geojson`);
  const data = await response.json();
  updateLayerData(layer.id, data);  // ← 回写 store
};
```

**修复 store 逻辑**:
```typescript
// ❌ 错误：layer.data 初始为 undefined，条件不满足
if (layer && layer.data) {
  layer.data = geojsonData;
}

// ✅ 正确：只检查 layer 是否存在
if (layer) {
  layer.data = geojsonData;
}
```

## 10. 总结

### 10.1 架构优势

1. **渲染策略分离**: PointRendererManager 统一调度，便于扩展
2. **数据驱动**: 数据回写 store，渲染器从 store 读取，解耦
3. **资源管理**: 独立的 cache 机制，避免重复加载
4. **精细化控制**: 支持单点覆盖配置（预留）

### 10.2 核心技术点

| 技术点 | 解决方案 |
|--------|----------|
| 二三维切换 | PointRenderMode + 渲染器工厂 |
| 自定义符号 | Canvas 动态生成 + 缓存 |
| 图片上传 | FileReader + dataURL |
| 文字标注 | LabelCollection + 属性字段解析 |
| 位置联动 | Label 使用 Billboard 相同位置 + 动态偏移 |
| 高度控制 | pointHeightOffset 替代 heightReference |

### 10.3 经验总结

1. **Entity API 误区**: `Entity.billboardCollection` 不存在，应直接使用 primitives
2. **heightReference 限制**: primitives 不支持，需用高度偏移替代
3. **清空顺序**: 必须先清空 collection 再添加，否则重叠
4. **数据流一致性**: 确保数据加载完成后再渲染，避免竞态
