# 任务清单

标准：不看架构和功能设计，不管是初级开发者还是不那么厉害的AI，照着操作能够基本复原出项目的基本功能。

# PHASE——1

## 1. 库结构初始化与整理

1.  [ ] **创建目录结构**
    - 在 `libs/features/map-core/src/lib` 下创建以下子目录：
      - `components/`
      - `renderer/`
      - `controllers/`
      - `store/`
      - `types/`
      - `hooks/` (已有则复用)
    - _说明_：确保目录结构符合架构设计，方便分层管理。

2.  [ ] **迁移/清理旧代码**
    - 检查 `libs/features/map-core/src/lib` 下现有的 `state/`, `components/MapViewer.tsx`, `hooks/`。
    - 将 `MapViewer.tsx` 拆分或重构为新架构下的组件。
    - 将 `map-state.ts` 移动到 `types/` 并更新定义。
    - _说明_：保持代码库整洁，避免新旧逻辑混杂。

## 2. 后端 API 开发 (GIS-DataManager)

3.  [ ] **实现 GeoJSON 服务接口**
    - 文件：`libs/features/GIS-DataManger/api/src/lib/dataset.service.ts`
    - 任务：添加 `getDatasetGeoJSON(id: string)` 方法。
    - 逻辑：
      - 根据 `id` 查找最新的 `DatasetVersion`。
      - 查询该版本关联的所有 `GisFeature`。
      - 将 `properties` (JSON) 和 `geometry` (PostGIS Geometry) 组装成 GeoJSON Feature。
      - 返回 `{ type: "FeatureCollection", features: [...] }`。
    - _提示_：使用 `ST_AsGeoJSON` 将 PostGIS 几何体转为 GeoJSON 字符串。

4.  [ ] **暴露 API 端点**
    - 文件：`libs/features/GIS-DataManger/api/src/lib/dataset.controller.ts`
    - 任务：添加 `GET /datasets/:id/geojson` 路由。
    - 逻辑：调用 `datasetService.getDatasetGeoJSON(id)` 并返回结果。

## 3. 前端核心逻辑 (Map Core)

5.  [ ] **定义类型与状态**
    - 文件：`libs/features/map-core/src/lib/types/map-state.ts`
    - 任务：定义 `LayerState`, `MapStateSchema` 等接口（参考架构文档）。

6.  [ ] **实现全局 Store**
    - 文件：`libs/features/map-core/src/lib/store/use-map-store.ts`
    - 任务：使用 Zustand 创建 Store。
    - 功能：
      - `layers`: 数组状态。
      - `addLayer(layer)`: 添加图层。
      - `removeLayer(id)`: 移除图层。
      - `updateLayer(id, updates)`: 更新图层属性（显隐、样式）。
      - `setViewport(viewport)`: 更新相机状态。

7.  [ ] **封装 Cesium 容器**
    - 文件：`libs/features/map-core/src/lib/renderer/map-container.tsx`
    - 任务：初始化 Cesium Viewer。
    - 逻辑：
      - 挂载时创建 `new Cesium.Viewer`。
      - 隐藏默认控件（BaseLayerPicker, Animation, Timeline 等）。
      - 将 viewer 实例暴露给父组件或通过 Context 共享（推荐使用 Zustand Store 存储 viewer 引用，或者通过 Props 传递给 Renderer）。

8.  [ ] **实现图层渲染器**
    - 文件：`libs/features/map-core/src/lib/renderer/layer-renderer.tsx`
    - 任务：响应 Store 变化并操作 Cesium。
    - 逻辑：
      - 监听 `useMapStore.layers`。
      - **新增图层**：调用后端 `/api/datasets/:id/geojson`，获取数据后使用 `Cesium.GeoJsonDataSource.load` 加载，添加到 `viewer.dataSources`。
      - **移除图层**：从 `viewer.dataSources` 中移除对应 DataSource。
      - **更新显隐**：设置 `dataSource.show = visible`。
      - **更新样式**：遍历 `dataSource.entities` 修改材质颜色/宽度。

9.  [ ] **实现 AI 桥接器**
    - 文件：`libs/features/map-core/src/lib/controllers/map-agent-bridge.tsx`
    - 任务：连接 Socket 与 Store。
    - 逻辑：
      - 使用 `useMapSocket` 监听 `ui_action`。
      - 根据 `intent` (ADD_LAYER, TOGGLE_LAYER 等) 调用 Store 的 Actions。
      - 监听 Store 变化，通过 `socket.emit('state_update')` 回传地图状态。

## 4. 前端 UI 组件 (Map Core)

10. [ ] **实现添加图层弹窗**
    - 文件：`libs/features/map-core/src/lib/components/add-layer-modal.tsx`
    - 任务：复用 `DatasetTable` 逻辑。
    - 逻辑：
      - 调用 `GET /api/datasets` 获取列表。
      - 选中后点击确定，调用 `store.addLayer`，传入 datasetId 和 name。

11. [ ] **实现图层管理器**
    - 文件：`libs/features/map-core/src/lib/components/layer-manager.tsx`
    - 任务：左侧悬浮面板。
    - 逻辑：
      - 从 Store 读取 `layers` 列表。
      - 渲染图层树：图标 + 名称 + 显隐 Checkbox + 删除按钮。
      - 操作绑定：Checkbox -> `store.updateLayer(id, { visible })`；删除 -> `store.removeLayer(id)`。

12. [ ] **实现样式面板**
    - 文件：`libs/features/map-core/src/lib/components/style-panel.tsx`
    - 任务：右侧属性面板。
    - 逻辑：
      - 展示当前选中图层的样式属性（颜色、宽度）。
      - 修改后实时调用 `store.updateLayer`。

13. [ ] **组装地图页面**
    - 文件：`libs/features/map-core/src/lib/components/map-viewer.tsx` (更新入口组件)
    - 任务：组合上述组件。
    - 结构：
      ```tsx
      <div className="relative w-full h-full">
        <MapContainer />
        <LayerRenderer />

        <div className="absolute top-4 left-4 z-10">
          <LayerManager />

        <StylePanel />

      ```

    - 文件：`apps/map-ai/web/src/pages/map/index.tsx` (或类似路由页面)

15. [ ] **全链路测试**

## 2. 绘图功能 (Draw Tools)

18. [ ] **实现绘图工具栏**
    - 文件：`libs/features/map-core/src/lib/components/draw-toolbar.tsx`
    - 任务：右侧悬浮工具栏。
    - 逻辑：
      - 渲染 3 个按钮：标点、标线、标面。
      - 点击按钮调用 `store.setInteractionMode('draw_point' | 'draw_line' | 'draw_polygon')`。
      - 高亮当前激活的工具。

19. [ ] **实现交互渲染器**
    - 文件：`libs/features/map-core/src/lib/renderer/draw-renderer.tsx`
    - 任务：处理 Cesium 交互事件。
    - 逻辑：
      - 监听 `store.interaction.mode`。
      - 使用 `Cesium.ScreenSpaceEventHandler` 监听左键点击、移动、右键点击。
      - **绘制逻辑**：
        - Point: 点击即完成，添加到 Store。
        - Line: 点击添加节点，移动预览，右键结束。
        - Polygon: 点击添加节点，移动预览，右键结束（闭合）。
      - **结果处理**：绘制完成后，生成 GeoJSON Feature，调用 `store.addLayer` (如果不存在 Draw 图层) 或添加到现有 Draw 图层。

## 3. 属性表与编辑 (Attribute Panel)

20. [ ] **实现属性面板**
    - 文件：`libs/features/map-core/src/lib/components/attribute-panel.tsx`
    - 任务：底部抽屉组件。
    - 逻辑：
      - 监听 `store.selection`。
      - 如果有选中要素，显示面板。
      - 渲染属性表格 (Key-Value)。
      - 支持编辑 Value：双击或点击编辑图标，输入新值，回车保存 -> 调用 `store.updateLayerFeature`。

21. [ ] **增强图层渲染器 (支持拾取)**
    - 文件：`libs/features/map-core/src/lib/renderer/layer-renderer.tsx`
    - 任务：添加点击拾取功能。
    - 逻辑：

      - 使用 `viewer.scene.pick(position)` 获取点击的 Entity。
      - 如果 pick 到 Entity，查找对应的 Layer ID 和 Feature ID。
      - 调用 `store.setSelection({ layerId, featureId, properties })`。

22. [ ] **更新 UI 入口**
