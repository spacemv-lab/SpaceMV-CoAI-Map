# 点图层样式问题分析与三维点样式设计

## 1. 问题描述

用户反馈：在样式设置面板中调整点图层的自定义样式时，**只有颜色可以调整成功**，其他属性如：
- 不透明度（opacity）
- 尺寸单位（pointSizeUnit）
- 尺寸大小（pointSize）
- 符号形状（pointSymbol）

均无法生效。

## 2. 问题原因分析

### 2.1 根本原因

经过代码分析，问题的根本原因是：**Cesium 中的点图层使用了 Billboard（广告牌）渲染，而不是 PointPrimitive**。

### 2.2 代码证据

在 `layer-renderer.tsx:247-260` 中：

```typescript
if (entity.point) {
  entity.point.color = new Cesium.ConstantProperty(
    color.withAlpha(opacity),
  );
  entity.point.pixelSize = new Cesium.ConstantProperty(
    layer.style.pointSize || 10,
  );
}
if (entity.billboard) {
  entity.billboard.color = new Cesium.ConstantProperty(
    color.withAlpha(opacity),
  );
  // Scale could be mapped to pointSize
}
```

可以看到：
1. **颜色（color）**：同时支持 `entity.point` 和 `entity.billboard`
2. **不透明度（opacity）**：通过 `color.withAlpha(opacity)` 应用到颜色上，所以**实际上应该生效**
3. **尺寸（pointSize）**：只支持 `entity.point`，**不支持** `entity.billboard`
4. **尺寸单位（pointSizeUnit）**：**完全未实现**
5. **符号形状（pointSymbol）**：**完全未实现**

### 2.3 为什么点图层使用 Billboard 而不是 Point？

查看 `layer-renderer.tsx:139-147` 的 GeoJSON 加载逻辑：

```typescript
dataSource = await Cesium.GeoJsonDataSource.load(data, {
  stroke: Cesium.Color.fromCssColorString(
    layer.style.color || '#3388ff',
  ),
  fill: Cesium.Color.fromCssColorString(
    layer.style.color || '#3388ff',
  ).withAlpha(layer.style.opacity ?? 0.5),
  strokeWidth: layer.style.width || 2,
});
```

**关键发现**：`Cesium.GeoJsonDataSource.load()` 方法**没有传递 `pointRadius` 或 `pointShape` 等点样式参数**。

根据 Cesium 官方文档，当 GeoJSON 包含点要素时，默认会创建 `Billboard` 而不是 `PointPrimitive`，因为：
- **Billboard**：支持图片/图标，适合标记点
- **PointPrimitive**：仅支持简单几何形状（圆形），性能更好但不支持自定义图标

### 2.4 为什么颜色能生效？

颜色能生效是因为代码中对 `billboard.color` 进行了处理：

```typescript
entity.billboard.color = new Cesium.ConstantProperty(
  color.withAlpha(opacity),
);
```

但这只是修改了 Billboard 的整体颜色（包括透明度），**Billboard 的其他属性如 scale、image 等并未根据样式配置更新**。

## 3. Cesium 支持的三维点渲染方式

### 3.1 PointPrimitive（点图元）

```typescript
entity.point = {
  pixelSize: 10,           // 像素大小
  color: Color.RED,        // 颜色
  outlineColor: Color.WHITE, // 轮廓颜色
  outlineWidth: 2,         // 轮廓宽度
  scaleByDistance: new NearFarScalar(...), // 根据距离缩放
  translucencyByDistance: new NearFarScalar(...), // 根据距离透明度
  heightReference: HeightReference.CLAMP_TO_GROUND, // 高度参考
  verticalOrigin: VerticalOrigin.CENTER, // 垂直原点
  horizontalOrigin: HorizontalOrigin.CENTER, // 水平原点
  disableDepthTestDistance: number, // 禁用深度测试距离
}
```

**特点**：
- 仅支持**圆形**点
- 性能优秀（适合大量点）
- 支持基于距离的 LOD（细节层次）
- 支持轮廓

### 3.2 Billboard（广告牌）

```typescript
entity.billboard = {
  image: './icon.png',     // 图片 URL 或 Canvas
  scale: 1.0,              // 缩放比例
  color: Color.RED,        // 颜色（乘法）
  opacity: 1.0,            // 透明度
  rotation: 0,             // 旋转（弧度）
  alignedAxis: Cartesian3.UNIT_Z, // 对齐轴
  horizontalOrigin: HorizontalOrigin.CENTER,
  verticalOrigin: VerticalOrigin.CENTER,
  heightReference: HeightReference.CLAMP_TO_GROUND,
  width: number,           // 固定宽度（像素）
  sizeInMeters: false,     // 是否使用米为单位
  eyeOffset: Cartesian3,   // 眼睛偏移
  disableDepthTestDistance: number,
}
```

**特点**：
- 支持**自定义图片/图标**
- 支持旋转
- 可选择像素或米为单位
- 性能较 PointPrimitive 差

### 3.3 Model（3D 模型）

```typescript
entity.model = {
  uri: './model.glb',      // glTF/GLB 模型路径
  scale: 1.0,              // 缩放
  minimumPixelSize: 64,    // 最小像素大小
  maximumScale: 200,       // 最大缩放
  color: Color.WHITE,      // 颜色
  silhouetteColor: Color.BLACK, // 轮廓颜色
  silhouetteSize: 2,       // 轮廓大小
}
```

**特点**：
- 完整的 3D 模型
- 支持动画
- 性能开销最大

## 4. 解决方案设计

### 4.1 短期方案：修复现有 Billboard 样式支持

修改 `layer-renderer.tsx` 的 `updateLayerStyle` 函数，增加对 Billboard 的完整样式支持：

```typescript
if (entity.billboard) {
  // 颜色 + 透明度
  entity.billboard.color = new Cesium.ConstantProperty(
    color.withAlpha(opacity),
  );

  // 尺寸（映射到 scale）
  entity.billboard.scale = new Cesium.ConstantProperty(
    layer.style.pointSize || 1.0
  );

  // 尺寸单位
  entity.billboard.sizeInMeters = new Cesium.ConstantProperty(
    layer.style.pointSizeUnit === 'meters'
  );

  // 符号形状（需要预定义图标或使用 Cesium 内置符号）
  // 方案：使用 SVG 动态生成或使用 Cesium 内置的 SymbolMaterial
}
```

**符号形状实现方案**：
1. 使用 Canvas 动态绘制符号（圆形、方形、三角形、星形、菱形）
2. 转换为DataURL 赋值给 `entity.billboard.image`

### 4.2 长期方案：提供 PointPrimitive 选项

在图层创建时，允许用户选择点渲染方式：

```typescript
interface LayerState {
  // ... 现有字段
  pointRenderMode?: 'billboard' | 'point' | 'model'; // 点渲染模式
}
```

### 4.3 三维点样式配置扩展

建议新增以下样式属性：

```typescript
export type LayerStyle = {
  // 通用属性
  color?: string;
  width?: number;
  opacity?: number;

  // 点样式专属
  pointSize?: number;
  pointSizeUnit?: 'pixels' | 'meters';
  pointSymbol?: 'circle' | 'square' | 'triangle' | 'star' | 'diamond' | 'custom';
  pointRenderMode?: 'billboard' | 'point'; // 渲染模式
  pointOutlineColor?: string; // 轮廓颜色（仅 PointPrimitive）
  pointOutlineWidth?: number; // 轮廓宽度（仅 PointPrimitive）
  pointRotation?: number; // 旋转角度（仅 Billboard）
  pointImageUri?: string; // 自定义图标（仅 Billboard）

  // ... 其他
}
```

## 7. 代码示例

### 7.1 Canvas 动态绘制符号

```typescript
/**
 * 根据符号类型生成 Billboard 图片
 */
function createSymbolCanvas(
  symbol: 'circle' | 'square' | 'triangle' | 'star' | 'diamond',
  color: string,
  size: number = 32
): HTMLCanvasElement {
  const canvas = document.createElement('canvas');
  canvas.width = size;
  canvas.height = size;
  const ctx = canvas.getContext('2d')!;

  ctx.fillStyle = color;
  ctx.beginPath();

  const center = size / 2;
  const radius = size / 2 - 2;

  switch (symbol) {
    case 'circle':
      ctx.arc(center, center, radius, 0, Math.PI * 2);
      break;
    case 'square':
      ctx.rect(2, 2, size - 4, size - 4);
      break;
    case 'triangle':
      ctx.moveTo(center, 2);
      ctx.lineTo(size - 2, size - 2);
      ctx.lineTo(2, size - 2);
      break;
    case 'diamond':
      ctx.moveTo(center, 2);
      ctx.lineTo(size - 2, center);
      ctx.lineTo(center, size - 2);
      ctx.lineTo(2, center);
      break;
    case 'star':
      // 五角星绘制逻辑
      break;
  }

  ctx.fill();
  return canvas;
}
```

### 7.2 更新图层样式

```typescript
const updateLayerStyle = (
  layer: LayerState,
  dataSource: Cesium.DataSource,
) => {
  const entities = dataSource.entities.values;
  for (const entity of entities) {
    const color = Cesium.Color.fromCssColorString(
      layer.style.color || '#3388ff',
    );
    const opacity = layer.style.opacity ?? 1;

    // PointPrimitive 样式更新
    if (entity.point) {
      entity.point.color = new Cesium.ConstantProperty(
        color.withAlpha(opacity),
      );
      entity.point.pixelSize = new Cesium.ConstantProperty(
        layer.style.pointSize || 10,
      );
      if (layer.style.pointOutlineColor) {
        entity.point.outlineColor = new Cesium.ConstantProperty(
          Cesium.Color.fromCssColorString(layer.style.pointOutlineColor),
        );
        entity.point.outlineWidth = new Cesium.ConstantProperty(
          layer.style.pointOutlineWidth || 1,
        );
      }
    }

    // Billboard 样式更新
    if (entity.billboard) {
      entity.billboard.color = new Cesium.ConstantProperty(
        color.withAlpha(opacity),
      );
      entity.billboard.scale = new Cesium.ConstantProperty(
        layer.style.pointSize || 1.0,
      );
      entity.billboard.sizeInMeters = new Cesium.ConstantProperty(
        layer.style.pointSizeUnit === 'meters',
      );

      // 动态生成符号图标
      if (layer.style.pointSymbol) {
        const symbolCanvas = createSymbolCanvas(
          layer.style.pointSymbol as any,
          layer.style.color || '#ffffff',
          64
        );
        entity.billboard.image = new Cesium.ConstantProperty(
          symbolCanvas.toDataURL('image/png')
        );
      }
    }
  }
};
```

## 8. 总结

1. **问题根源**：Cesium GeoJsonDataSource 默认使用 Billboard 渲染点，但样式更新代码只处理了 PointPrimitive 的大部分属性

2. **颜色为何生效**：Billboard 的 color 属性被正确更新

3. **解决方案**：
   - 短期：完善 Billboard 的样式支持（scale、sizeInMeters、动态图标）
   - 长期：提供 PointPrimitive 渲染模式选项

4. **推荐三维点样式**：简单点、标记点、图标点、3D 点、标签点 五种基础类型

5. **实施建议**：先修复 Bug，再增强功能，最后实现高级特性
