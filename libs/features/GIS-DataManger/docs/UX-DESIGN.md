## 1. 用户旅程

### 1.1 主流程

```
进入数据管理页面
    ↓
查看数据列表（空状态/已有数据）
    ↓
点击「上传数据」按钮
    ↓
选择文件（拖拽或点击）
    ↓
显示上传进度
    ↓
后台解析（异步）
    ↓
上传成功，列表刷新
    ↓
查看/使用数据
```

### 1.2 异常处理

| 场景 | 处理方式 |
|------|----------|
| 网络错误 | 显示重试按钮，保留已选文件 |
| 文件格式不支持 | 上传前校验，提示支持的格式 |
| 文件过大 | 进度条显示失败，提示大小限制 |
| 解析失败 | 显示错误原因，提供降级方案 |
| 数据为空 | 显示空状态插图和操作引导 |

## 2. 交互规范

### 2.1 反馈机制

| 操作 | 反馈 |
|------|------|
| 点击上传按钮 | 打开上传模态框 |
| 拖拽文件进入 | 高亮 Dropzone 区域 |
| 文件选择完成 | 显示文件名和大小 |
| 上传开始 | 显示进度条（0% - 100%） |
| 上传成功 | Toast 提示 + 列表刷新 |
| 上传失败 | Toast 错误提示 + 重试按钮 |
| 点击删除 | 弹出确认对话框 |
| 删除完成 | Toast 提示 + 列表刷新 |

### 2.2 加载状态

- **首次加载**：骨架屏（Skeleton）
  ```tsx
  <Skeleton className="h-32 w-full" />
  ```

- **分页加载**：底部加载指示器
  ```tsx
  {isLoading && <Spinner />}
  ```

- **操作中**：按钮禁用 + 加载动画
  ```tsx
  <Button disabled={isUploading}>
    {isUploading && <Loader2 className="animate-spin" />}
    上传中...
  </Button>
  ```

## 5. 用户体验优化点

### 5.1 上传体验

1. **拖拽上传**：支持拖拽文件到指定区域
2. **格式提示**：清晰展示支持的格式列表
3. **进度反馈**：实时显示上传和解析进度
4. **后台处理**：大文件异步处理，不阻塞界面

### 5.2 数据展示

1. **智能分页**：默认每页 10 条，支持切换
2. **列排序**：点击表头按列排序
3. **快捷操作**：常用操作（详情、下载、删除）一键触达
4. **状态可视**：解析状态用 Badge 清晰标识

### 5.3 错误恢复

1. **失败重试**：上传失败可一键重试
2. **操作撤销**：删除后短时间内可撤销（可选）
3. **错误定位**：清晰提示错误原因和解决建议
